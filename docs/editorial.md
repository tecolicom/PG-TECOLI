# 週刊鉄塔マガジン 編集部の手引き

「週刊鉄塔マガジン」は [鉄塔てこり](https://pg.tecoli.com) の看板コンテンツ（路線ツアー・見どころ）を
発行する**公開編集部**です。このリポジトリの Issue・Milestone・ラベルを使って、
起案から公開までの編集作業を GitHub 上でオープンに行います。

## 編集部の紹介

- **編集長** = このリポジトリのオーナー。採用の決定と公開の GO を持ちます
- **執筆担当** = AI（Claude Code のセッション巡回）。Issue やコメントには
  末尾に「（執筆担当 AI）」と署名して投稿し、人格を区別します
- **編集部員** = GitHub アカウントを持つ誰でも。起案やコメントで参加できます

## 記事のライフサイクル

記事は次の6段階を、Issue に付いたラベルの変化としてたどります。

1. **起案** — 記事テンプレート「[記事の起案](../.github/ISSUE_TEMPLATE/article-proposal.md)」で
   Issue を立てる（執筆担当・編集長・編集部員の誰が立ててもよい）。ラベル `起案`
2. **採用** — 編集長が採用を決定し、ラベル `採用` を付けて号の Milestone（例: 「第4号」）を割り当てる
3. **執筆中** — 執筆担当がブランチ `draft/NNN-<id>` に YAML を push（`build.py` 実行済みで
   `tours.json` も込み）。ラベル `執筆中` に切り替え、Issue にプレビュー URL をコメントする
4. **校正** — 編集部員が Issue にコメントし、執筆担当が修正を push（URL は変わらず、リロードで最新になる）。
   ラベル `校正`
5. **採用（マージ）** — 編集長の OK で `main` へマージ。この時点ではまだ `published` は付かず、
   「ストック」として main に入るだけで本番には出ない
6. **公開（週次リリース）** — 発行のタイミングで編集長が該当ツアー YAML に `published: true` を付けて
   main へコミットし、10map 側で `make tours home` を実行して本番デプロイする。
   ラベルを `公開済み` にして Issue をクローズする

## 号（Milestone）と記事（Issue）

- **号 = Milestone**。「第N号」として号ごとに立て、採用された記事の Issue を割り当てます
- **記事 = Issue**。1記事につき1 Issue。起案から公開までの状態はラベルで管理します
- 種別ラベル `ツアー` / `見どころ` で記事の種類も分かるようにしています

## プレビューの見かた

執筆中・校正中の Issue には、次の形式のプレビュー URL がコメントとして貼られます。

```
https://pg.tecoli.com/tour/preview?data=tecolicom/PG-TECOLI@<ブランチ>&id=<記事id>&issue=<N>
```

このリンクを開くと、ドラフトブランチの内容を実際のツアーページと同じ見た目で確認できます。
修正が push されたら、ページをリロードすれば最新の内容が反映されます（URL は変わりません）。

## コメントのしかた

記事を読んで、Issue に自由にコメントしてください。表現・事実関係・構成、なんでも歓迎です。
採用・不採用の決定と、公開するかどうかの判断は編集長が行います。

## 執筆担当向けの技術メモ

- ドラフトはブランチ `draft/NNN-<id>` に作成する
- YAML は `tours/` の規約（README 参照）どおりに書く
- **push 前に必ず `python3 scripts/build.py` を実行し、生成された `tours.json` もコミットする**
  （プレビューページは `tours.json` を読むため、これを忘れるとプレビューに反映されません）
